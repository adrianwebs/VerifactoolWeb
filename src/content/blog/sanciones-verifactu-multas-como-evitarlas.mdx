## El régimen sancionador de Verifactu

El **Real Decreto 1007/2023** y la **Ley 11/2021 de medidas de prevención y lucha contra el fraude fiscal** (conocida como la Ley Antifraude) establecen un régimen sancionador claro y contundente para quienes no cumplan con los requisitos de Verifactu.

Las sanciones afectan a **dos colectivos diferentes**:
- Los **usuarios** del software (autónomos, pymes, empresas).
- Los **fabricantes y distribuidores** del software.

## Sanciones para usuarios del software

### Uso de software no homologado

Si como autónomo o empresa utilizas un programa de facturación que **no cumple los requisitos técnicos** de Verifactu:

| Infracción | Sanción |
|-----------|---------|
| Uso de software no certificado | **Hasta 50.000 € por ejercicio fiscal** |
| Uso de software con funciones de doble uso | **Hasta 150.000 €** |
| Manipulación de registros de facturación | **Hasta 150.000 € + recargos** |

### ¿Qué es el "doble uso"?

El concepto de **software de doble uso** se refiere a programas que permiten:
- Llevar una **doble contabilidad** (una oficial y una "B").
- **Eliminar o modificar** registros de facturación ya emitidos.
- **Ocultar facturas** o ventas al sistema.

La mera posesión de un software con estas capacidades ya es sancionable, incluso si no se ha utilizado para defraudar.

## Sanciones para fabricantes de software

Los fabricantes y distribuidores de software de facturación se enfrentan a sanciones aún más severas:

### Distribución de software no conforme

| Infracción | Sanción |
|-----------|---------|
| Fabricar/distribuir software no conforme | **150.000 € por ejercicio fiscal** |
| Software con funciones de doble uso | **150.000 € por ejercicio** |
| No incluir declaración responsable | **1.000 € por declaración** |

### Declaración responsable obligatoria

Todos los fabricantes de software deben emitir una **declaración responsable** certificando que su programa cumple con los requisitos del reglamento. Esta declaración debe acompañar al software y estar disponible para los usuarios.

## ¿Cómo se detectan las infracciones?

La AEAT tiene múltiples mecanismos para detectar el incumplimiento:

### 1. Cruce de datos automático
Con Verifactu, la AEAT recibirá los registros de facturación en tiempo real. Los **huecos en la cadena de hash** o las **inconsistencias** entre lo declarado y lo registrado son detectadas automáticamente.

### 2. Inspecciones tributarias
Los inspectores pueden solicitar acceso al sistema de facturación y verificar:
- Que el software tiene la certificación Verifactu.
- Que la cadena de hash está intacta.
- Que no se han eliminado registros.

### 3. Verificación QR
Cualquier persona (cliente, proveedor, competidor) puede escanear el **código QR** de una factura y verificar su autenticidad con la AEAT. Si la factura no está registrada, se activa una alerta.

### 4. Denuncias de terceros
La AEAT permite que terceros reporten facturas sospechosas, lo que añade una capa adicional de control social.

## Casos reales: ejemplos de infracciones

### Caso 1: El autónomo que "no sabía"
Un fontanero autónomo sigue usando un programa de facturación antiguo que no cumple con Verifactu. Aunque no tiene intención de defraudar, la mera utilización del software no homologado después del 1 de julio de 2026 le expone a una sanción de **hasta 50.000 €**.

### Caso 2: La pyme con software pirata
Una pequeña empresa de construcción utiliza un software que permite eliminar facturas. Aunque solo lo usaban para "corregir errores", el programa tiene funciones de doble uso. Sanción potencial: **150.000 €**.

### Caso 3: El asesor que recomienda mal
Un asesor fiscal recomienda a sus clientes un software que no cumple. Aunque el asesor no es el fabricante, los clientes que lo usen se enfrentan a sanciones. La reputación del asesor queda destruida.

## Cómo evitar las sanciones: checklist

Para asegurarte de que tu negocio cumple y evitar cualquier sanción:

### ✅ Checklist de cumplimiento Verifactu

1. **¿Tu software tiene declaración responsable del fabricante?** Si no la tiene, cambia de software.
2. **¿Genera hash encadenado?** Cada factura debe estar vinculada criptográficamente a la anterior.
3. **¿Envía registros a la AEAT?** El envío debe ser automático (opción Verifactu) o almacenarse para inspección (opción no Verifactu).
4. **¿Incluye QR verificable?** Todas las facturas deben llevar un código QR que enlace al sistema de verificación de la AEAT.
5. **¿Impide la manipulación?** El software no debe permitir eliminar, modificar o reordenar facturas emitidas.
6. **¿Dispone de registro de eventos?** Debe registrar todas las acciones realizadas sobre los registros de facturación.
